## Why

当前博客的文章详情页底部展示的"上一篇/下一篇"导航基于全局时间顺序，这在从首页或文章列表页进入时是合理的。但当用户从标签页面（如 `/tags/astro`）进入文章时，导航显示的是全局时间顺序的上下篇，而不是该标签下的上下篇，这不符合用户直觉。

为了兼顾框架逻辑的精简性以及符合用户操作直觉，需要根据来源上下文决定是否展示导航：从首页/列表页进入时显示导航，从标签页进入时隐藏导航。

## What Changes

- **修改** 文章详情页的上一篇/下一篇导航逻辑，增加来源判断
- **新增** 基于来源类型的导航可见性规则：
  - `from=home` 或 `from=blog-list` → 显示导航
  - `from=tag-detail` → 隐藏导航
  - 无来源参数或无法识别 → 默认显示导航（等同于首页逻辑）
- **保持** 导航的时间排序逻辑不变（仍然基于全局时间顺序）
- **保持** 来源信息传递机制不变（`from` 和 `ref` 参数继续传递）

## Capabilities

### New Capabilities

无新增能力。

### Modified Capabilities

- **prev-next-nav**: 修改上一篇/下一篇导航的需求，增加基于来源类型的可见性控制规则。原有需求（时间排序、布局、样式、静态生成、来源信息保持）保持不变，新增"仅在特定来源下显示"的需求。

## Impact

- **受影响的组件**:
  - `PrevNextNav.astro`（或相应的导航组件）
  - 文章详情页模板

- **受影响的页面**:
  - `/blog/[slug]` 文章详情页

- **向后兼容性**:
  - 完全兼容，无破坏性变更
  - 现有的来源参数传递机制保持不变
  - 仅影响导航的可见性，不影响其他功能

- **依赖关系**:
  - 依赖 `smart-back-button` 中定义的来源类型（`from` 参数值）
  - 来源类型包括：`home`、`blog-list`、`tag-detail` 等
