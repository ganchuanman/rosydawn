Git 是现代软件开发不可或缺的工具。但很多人只用到了它功能的冰山一角，其实它隐藏着许多强大的特性。

## 提交的哲学

好的提交信息是一门艺术。遵循 Conventional Commits 规范，使用 `feat`、`fix`、`docs` 等前缀，让提交历史清晰可读。每次提交只做一件事，保持原子性，方便回溯和 cherry-pick。

## 分支策略

合理的分支策略能让团队协作更加顺畅。无论是 Git Flow 还是 Trunk-Based Development，选择适合团队的方式最重要。定期清理过时的分支，保持仓库整洁。

版本控制不只是备份代码，更是记录项目演进的编年史。用心对待每一次提交，未来的自己会感谢现在的你。
